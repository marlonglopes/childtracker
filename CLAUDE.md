# ChildTracker — Claude Code Context

## What This App Does
Transparent iOS parental monitoring. The child installs it knowingly and approves
a system DNS profile once. Every domain the child's phone queries is logged and
forwarded to the parent via WhatsApp. The original "child taps activity buttons
+ location" concept was scrapped — see commit `ef5c9d6`.

## Tech Stack
- **App**: React Native via Expo. Currently running on **Expo-managed** for
  Sprint 1 (no `ios/` dir yet). Sprint 2 introduces `expo prebuild` so the
  Network Extension target can be added via an Expo config plugin.
- **Language**: TypeScript (app + backend), Swift (Network Extension, Sprint 2)
- **State**: Zustand + AsyncStorage (persisted)
- **Navigation**: React Navigation v6
- **Styling**: NativeWind (Tailwind for RN) — requires `react-native-css-interop`
  installed for types to resolve
- **Backend**: Firebase Firestore + Cloud Functions (Node 20)
- **Messaging**: Twilio WhatsApp (mocked via logger when
  `FUNCTIONS_EMULATOR=true`)
- **iOS Extension (Sprint 2)**: `NEDNSProxyProvider`

## Key Constraints
- Network Extension cannot be tested on a real device without an Apple Developer
  account ($99/yr)
- Expo-managed workflow is fine for Sprint 1. Sprint 2 requires `expo prebuild`
  because Network Extension entitlements need a real Xcode project.
- App Group entitlement required for shared storage between app and extension
- The child must explicitly tap "Allow" to install the VPN profile — iOS
  requirement, not optional

## Data Flow
1. `NEDNSProxyProvider` (Swift, Sprint 2) intercepts a DNS query and appends
   `{ domain, timestamp, appBundleId }` to the shared App Group `UserDefaults`
   buffer.
2. Main app background task drains the buffer every 60s and calls
   `dnsLogService.uploadBatch(familyId, childId, logs)`.
3. `onDnsLogBatch` Cloud Function evaluates each entry against
   `flaggedDomains` / `blockedDomains`, batch-writes to `dnsLogs`, and emits an
   instant WhatsApp message for matches (unless `alertMode === 'digest'`).
4. `hourlyDigest` cron aggregates the last hour of flagged logs for families in
   `digest` or `both` mode.

## Running Locally
```bash
npm install
npm run emulators         # Firebase emulators: Firestore 8080, Auth 9099, Functions 5001, UI 4000
npm run dev               # Emulators + Expo dev server concurrently
npm run typecheck         # tsc --noEmit (0 errors expected)
```

To send a fake DNS batch against the emulator for smoke testing:
```bash
curl -X POST http://localhost:5001/<projectId>/us-central1/onDnsLogBatch \
  -H 'Content-Type: application/json' \
  -d '{"familyId":"<id>","childId":"<id>","logs":[{"domain":"tiktok.com","timestamp":'$(date +%s%3N)'}]}'
```

## What Needs an Apple Developer Account (Sprint 2)
- Network Extension entitlement
- App Group entitlement
- Provisioning profiles for both targets (`ChildTracker` + `ChildTrackerDNS`)
- Real-device testing of DNS interception (simulator does not route real DNS
  through `NEDNSProxyProvider`)
