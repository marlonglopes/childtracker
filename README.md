# ChildTracker

Transparent iOS parental monitoring. The child installs the app openly and
approves a system DNS profile once; every domain their phone queries is logged
and forwarded to the parent via WhatsApp. No hidden capture, no location
tracking.

> **Status:** Sprint 1 (backend + family linking) nearly complete. Sprint 2
> (Network Extension) requires an Apple Developer account and begins 2026-05-02.
> See [`SPRINT.md`](./SPRINT.md) and [`ROADMAP.md`](./ROADMAP.md).

## How it works

1. A Swift `NEDNSProxyProvider` Network Extension intercepts every DNS query on
   the child's device and writes `{ domain, timestamp, appBundleId }` to a
   shared App Group buffer. *(Sprint 2 — placeholder module today.)*
2. The main app drains the buffer every 60s and POSTs the batch to the
   `onDnsLogBatch` Cloud Function.
3. The function writes to Firestore and — for any domain matching the family's
   flagged or blocked list — sends an instant WhatsApp message via Twilio.
4. A pub/sub cron (`hourlyDigest`) summarises the last hour of flagged activity
   for families on digest mode.

Full design: [`ARCHITECTURE.md`](./ARCHITECTURE.md).

## Getting started

```bash
# 1. install deps
npm install
(cd functions && npm install)

# 2. start the local stack (Firebase emulators + Expo dev server)
npm run dev

# alternatively, run them separately:
npm run emulators     # Firestore 8080, Auth 9099, Functions 5001, UI 4000
npm start             # Expo Metro bundler
```

The emulator runs without Firebase or Twilio accounts. WhatsApp messages are
replaced with `functions.logger.info` entries when
`FUNCTIONS_EMULATOR=true` — check the emulator logs to see the alert that
would have been sent.

### Smoke-test the backend

```bash
curl -X POST http://localhost:5001/<projectId>/us-central1/onDnsLogBatch \
  -H 'Content-Type: application/json' \
  -d '{
        "familyId": "<familyId>",
        "childId":  "<childId>",
        "logs": [
          { "domain": "tiktok.com", "timestamp": '"$(date +%s%3N)"' }
        ]
      }'
```

Add `tiktok.com` to the family's `flaggedDomains` in the Emulator UI first
to see an alert fire.

|---|---|
| `npm start` | Expo Metro bundler |

| `npm run functions:deploy` | Deploy Cloud Functions to Firebase |

```

├── app.config.ts            # Expo config
├── firebase.json            # Emulator + deploy config
├── firestore.rules          # dnsLogs are functions-only write
├── functions/src/index.ts   # onDnsLogBatch + hourlyDigest
├── src/
│   ├── navigation/          # AuthStack, ParentStack, ChildStack, RootNavigator
│   ├── screens/             # Welcome, ParentSetup, LinkCode, Dashboard, Settings, Monitor
│   ├── services/            # firebase, familyService, dnsLogService, extensionBridge
│   ├── store/               # authStore, familyStore (persisted via AsyncStorage)
│   └── types/               # dns, family, index
├── ARCHITECTURE.md
├── ROADMAP.md
├── SPRINT.md
└── CLAUDE.md
```

The native `ios/` directory does not exist yet. It will be generated by
`expo prebuild` at the start of Sprint 2, at which point the `ChildTrackerDNS`
Network Extension target is added via a config plugin.
